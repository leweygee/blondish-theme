The NRG Zine is BLOND:ISH's community-driven editorial platform living at `/journal/` on blondish.world. ## 4. LLMO (Large Language Model Optimization) Strategy

### Why LLMO Matters for BLOND:ISH

AI systems (ChatGPT, Gemini, Perplexity, Claude) are increasingly where fans discover music, plan festival trips, and research genres. When someone asks an AI "what is organic house music?" or "best female DJs," the content that gets cited is:

1. **Clearly structured** with headers, lists, and definitions
2. **Factually authoritative** with specific data, dates, and names
3. **Entity-rich** — mentions specific people, places, labels, tracks
4. **Fresh** — updated within the last 3 months
5. **Widely cited** — linked from other authoritative sources

BLOND:ISH's existing genre guides are **already structured perfectly for LLMO**. The gap is getting them indexed and building the citation network.

#### A. Content Structure (Already Strong — Maintain)

- [x] Bold key terms and entity names
- [x] Internal cross-linking between guides

#### C. Entity Optimization
Every NRG Zine article should reinforce BLOND:ISH's entity associations:
- **Primary entities:** BLOND:ISH, Vivie-Ann Bakos, organic house, Abracadabra, Bye Bye Plastic
- **Associated entities:** Ibiza, Tulum, Burning Man, afro house, melodic house, sustainability
- **Competitor entities to co-rank with:** Black Coffee, Bedouin, Lee Burridge, Damian Lazarus

Maintain **consistent naming** across all content:
- Always "BLOND:ISH" (not "Blondish" or "Blond:ish") in article body
- Always "Vivie-Ann Bakos" for the person behind the artist name
- Always link to internal pages when mentioning brand properties

#### D. Schema Markup for Articles
Every zine article should include `Article` schema with:
```json
{
  "@type": "Article",
  "headline": "...",
  "author": { "@type": "Person", "name": "..." },
  "publisher": { "@type": "Person", "name": "BLOND:ISH" },
  "datePublished": "...",
  "dateModified": "...",
  "about": [
    { "@type": "MusicGenre", "name": "organic house" },
    { "@type": "Thing", "name": "electronic music" }
  ],
  "mentions": [
    { "@type": "Person", "name": "BLOND:ISH", "sameAs": "https://blondish.world/about/" }
  ]
}
```

Plus `FAQPage` schema for every FAQ section.

#### E. Freshness Strategy
LLMs deprioritize content older than ~3 months. Strategy:
- **Quarterly refresh** of all Tier 1 genre guides (update track lists, add new artists, refresh stats)
- **Seasonal updates** for city/festival guides (pre-Ibiza season, pre-Burning Man, etc.)
- **dateModified** always updated in schema when content is refreshed
- **Changelog** at bottom of evergreen articles: "Last updated: [date] — Added [what changed]"

## 7. Automation Pipeline

### Content Production Automation

#### WordPress Publishing Pipeline
Since this is a WordPress block theme, use the WP REST API to automate publishing:

```bash
# Example: Publish markdown content via WP-CLI
wp post create \
  --post_type=post \
  --post_title="Article Title" \
  --post_content="$(cat content/journal/article.md | pandoc -f markdown -t html)" \
  --post_status=publish \
  --post_date="2025-11-15 10:00:00" \
  --post_category="Sound Lab" \
  --meta_input='{"_yoast_wpseo_focuskw":"target keyword"}'
```

#### Automated Workflows (n8n / WP Cron)

1. **Content Scheduling Pipeline**
   - Markdown files in `content/journal/` → parsed for frontmatter → scheduled in WordPress via REST API
   - Automated meta description, OG image, and schema generation from frontmatter
   - Social media auto-posting on publish (Twitter, Instagram story link)

2. **SEO Monitoring Automation**
   - Weekly Ahrefs position tracking for all target keywords
   - Automated alerts when a page drops out of top 10
   - Monthly content freshness audit — flag articles not updated in 90+ days

3. **Community Submission Pipeline**
   - Form submission → moderation queue → editorial review → publish workflow
   - Auto-acknowledgment emails to submitters
   - Scheduled "Community Roundup" posts from approved submissions

### Technical SEO Automation

1. **Internal Linking Bot**
   - Scan all zine articles for opportunities to cross-link
   - Maintain a keyword → URL mapping for automated internal link suggestions
   - Run monthly to catch new linking opportunities as content grows

2. **Schema Generation**
   - Auto-generate Article + FAQPage schema from markdown frontmatter
   - Validate schema on publish via Google's Rich Results Test API
   - Monitor Search Console for schema errors

3. **Sitemap & Indexing**
   - Auto-submit new URLs to Google/Bing on publish
   - Monitor index status via Search Console API
   - Flag and fix any indexing issues within 24 hours

4. **llms.txt Auto-Update**
   - On new article publish, automatically append to llms.txt
   - Maintain section structure (genre guides, scene reports, etc.)
   - Rebuild llms-full.txt with complete content monthly

## 11. Backlink & Citation Strategy

### Earning Links Naturally
1. **Definitive guides** earn links from forums, Reddit, and other blogs referencing them
